### Prerequisites
1. **OpenVPN Installed**  
   Ensure OpenVPN is installed on your system. You can install it with:
   ```bash
   sudo apt update
   sudo apt install openvpn
   ```

2. **VPN Configuration File**  
   Place your TryHackMe VPN configuration file (e.g., `5pyD3R.ovpn`) in the directory `/etc/openvpn/client/`. You can download this configuration file from your TryHackMe profile.

---

### Steps to Create the `thm.openvpn` Service

1. **Create the Service File**  
   Open a new file for the service definition:
   ```bash
   sudo nano /etc/systemd/system/thm.openvpn.service
   ```

   Add the following content:
   ```ini
   [Unit]
   Description=OpenVPN Service for TryHackMe VPN
   After=network.target

   [Service]
   Type=simple
   ExecStart=/usr/sbin/openvpn --config /etc/openvpn/client/5pyD3R.ovpn
   Restart=on-failure
   ExecReload=/bin/kill -HUP $MAINPID
   KillSignal=SIGTERM

   [Install]
   WantedBy=multi-user.target
   ```

2. **Reload `systemd` Daemon**  
   Reload the `systemd` manager to recognize the new service:
   ```bash
   sudo systemctl daemon-reload
   ```

3. **Start the Service**  
   To start the service immediately:
   ```bash
   sudo systemctl start thm.openvpn
   ```

4. **Enable the Service on Boot**  
   To ensure the VPN starts on boot:
   ```bash
   sudo systemctl enable thm.openvpn
   ```

5. **Check the Service Status**  
   Verify if the service is running successfully:
   ```bash
   sudo systemctl status thm.openvpn
   ```

---

### Managing the Service

| Command                                  | Description                        |
|------------------------------------------|------------------------------------|
| `sudo systemctl start thm.openvpn`       | Start the VPN service manually.   |
| `sudo systemctl stop thm.openvpn`        | Stop the VPN service.             |
| `sudo systemctl restart thm.openvpn`     | Restart the VPN service.          |
| `sudo systemctl status thm.openvpn`      | Check the status of the service.  |
| `sudo systemctl enable thm.openvpn`      | Enable the service at boot.       |
| `sudo systemctl disable thm.openvpn`     | Disable the service at boot.      |

---

### Logs and Troubleshooting

- View service logs with:
  ```bash
  journalctl -u thm.openvpn
  ```

- Verify the OpenVPN connection:
  ```bash
  ip addr
  ```

---

### Notes
- Replace `5pyD3R.ovpn` with the name of your actual TryHackMe VPN configuration file if different.
- Ensure proper permissions for the configuration file:
  ```bash
  sudo chmod 600 /etc/openvpn/client/5pyD3R.ovpn
  ```

This setup allows you to manage the TryHackMe VPN connection easily and ensure it starts automatically when the system boots.
